# 7. Step-by-Step: Your First Feature

## Scenario: Add a "Clear All" Button

### Step 1: Setup
```bash
# Clone the repo (if you haven't)
git clone https://github.com/rishabhrd09/vitaltrack.git
cd vitaltrack

# Make sure you're on main and up to date
git checkout main
git pull origin main

# Create your feature branch
git checkout -b feature/clear-all-button
```

### Step 2: Make Changes
```bash
# Edit the file you need to change
code vitaltrack-mobile/app/(tabs)/inventory.tsx

# Add your "Clear All" button code
```

### Step 3: Test Locally

**Terminal 1 - Start Backend:**
```bash
cd vitaltrack-backend
docker-compose up --build

# Wait for:
# "Database tables created/verified"
# "Uvicorn running on http://0.0.0.0:8000"
```

**Terminal 2 - Start Mobile:**
```bash
cd vitaltrack-mobile
npm install  # Only needed first time
npx expo start --clear
```

**On Your Phone:**
1. Open Expo Go app
2. Scan QR code from terminal
3. Test your changes

### Step 4: Commit and Push
```bash
git add .
git commit -m "feat: add clear all button to inventory"
git push origin feature/clear-all-button
```

### Step 5: Create Pull Request
1. Go to https://github.com/rishabhrd09/vitaltrack
2. You'll see a yellow banner: "feature/clear-all-button had recent pushes"
3. Click "Compare & pull request"
4. Fill in description
5. Click "Create pull request"

### Step 6: Wait for CI/CD
- Watch the checks running on your PR
- All must pass (green checkmarks)
- Download Preview APK from Expo to test

### Step 7: Get Review and Merge
- Request review from lead
- Address feedback if any
- Merge when approved

## When to Test What

| Test Type | When | How | Where |
|-----------|------|-----|-------|
| Unit Tests | Every PR | Automated by CI | GitHub Actions |
| Integration Tests | Every PR | Automated by CI | GitHub Actions |
| Preview APK Test | Before merge | Manual | Your phone |
| Production Test | After merge | Manual | Production app |

## Preview APK Testing Checklist

```
Before approving any PR, test these on Preview APK:

[ ] App launches without crash
[ ] New feature works as expected
[ ] Existing features still work (regression)
[ ] Works offline (airplane mode)
[ ] Works with slow network (throttle)
[ ] No UI glitches
[ ] Error messages make sense
```

# 9. Common Questions Answered

## Q1: "I pushed to my feature branch but nothing happened?"

**A:** Correct! CI/CD only runs when you:
- Create a Pull Request, OR
- Push to main branch

Just pushing to a feature branch does nothing automatically.

---

## Q2: "Where do I download the Preview APK?"

**A:** Three ways:
1. **Expo Dashboard**: https://expo.dev â†’ Your project â†’ Builds
2. **GitHub Actions**: Check the "Build Preview APK" job logs for download link
3. **Command Line**: `npx eas build:list` then find URL

---

## Q3: "CI passed but I can't find the APK?"

**A:** Check if "Build Preview APK" job actually ran:
- On PRs: It should run âœ…
- On main pushes: It's skipped âŠ˜ (only Production AAB builds)

---

## Q4: "What's the difference between APK and AAB?"

**A:**
| Format | Use Case | Install Method |
|--------|----------|----------------|
| APK | Testing | Direct download and install |
| AAB | Play Store | Upload to Google Play Console |

---

## Q5: "My PR tests passed but Railway didn't deploy?"

**A:** Correct behavior! Railway only deploys when:
- Code is merged to `main` branch
- PRs only run tests, no deployment

---

## Q6: "How do I test backend changes before merge?"

**A:** Backend changes can only be tested:
1. **Locally**: Using Docker (`docker-compose up`)
2. **Staging environment**: If you set one up on Railway

Preview APK connects to production backend, so backend PR changes aren't testable there.

---

## Q7: "Can I run CI/CD manually?"

**A:** Yes! On GitHub:
1. Go to Actions tab
2. Select your workflow
3. Click "Run workflow"
4. Choose branch
5. Click "Run"

---

## Q8: "What if CI fails?"

**A:** 
1. Click on the failed job
2. Read the error logs
3. Fix the issue locally
4. Commit and push again
5. CI re-runs automatically on your PR
